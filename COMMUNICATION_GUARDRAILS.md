# Communication Guardrails

These guardrails apply to all commit messages, PR descriptions, docs, and release notes in this repository.

## Rules

- Do not mention internal tooling origins.
- Do not mention any assistant, generation workflow, or editor-specific creation source.
- Keep wording product-focused and implementation-focused only.
- Use neutral, professional engineering language.

## Commit Message Policy

- Describe what changed and why.
- Keep context tied to product goals and technical outcomes.
- Avoid references to how code was authored.
